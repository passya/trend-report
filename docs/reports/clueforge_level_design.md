## 1. 난이도 진행 곡선

### 1.1 난이도별 파라미터

| 파라미터 | 😊 쉬움 | 😐 보통 | 😰 어려움 | 💀 악몽 |
|----------|:------:|:------:|:--------:|:------:|
| 용의자 수 | 4 | 6 | 6 | 8 |
| 동기 보유자 | 2 | 3 | 4~5 | 전원 |
| 총 단서 | 6 | 10 | 12 | 15 |
| 레드 헤링 | 0 | 2 | 3~4 | 5~6 |
| 결정적 단서 | 1 | 2 | 3+ | 4+ |
| 장소 수 | 4 | 6 | 6 | 8 |
| NPC 감정 민감도 | 낮음 | 보통 | 높음 | 극높음 |
| 실언 유도 난이도 | 쉬움 (3턴) | 보통 (5턴) | 어려움 (7턴) | 극어려움 (10턴) |
| 목표 해결 시간 | 15~20분 | 30~40분 | 40~60분 | 60~90분 |
| 해결률 목표 | 90% | 70% | 50% | 25% |

### 1.2 신규 유저 난이도 경험 설계

```
사건 #1~3:   쉬움 (강제) + 튜토리얼 힌트 ON
사건 #4~6:   쉬움/보통 선택 가능
사건 #7~10:  보통 (기본), 쉬움/어려움 선택 가능
사건 #11~25: 전 난이도 해금
사건 #25+:   악몽 난이도 해금 (메타 진행 보상)
```

## 2. 세션 페이싱 (감정 곡선)

### 2.1 이상적 세션 감정 흐름 (보통 난이도, 35분)

```
감정 강도
  ↑
5 │                                    ★ 해설/카타르시스
  │                                   ╱
4 │                    ⚡실언!        ╱
  │                   ╱    ╲       ╱
3 │          첫 심문 ╱      ╲    ╱ 확신 고발
  │         ╱      ╱        ╲ ╱
2 │ 수색  ╱      ╱   증거보드 작업
  │     ╱      ╱     (사고 구간)
1 │ 도착╱   추가수색
  │   ╱
0 ├───┬───┬───┬───┬───┬───┬───┬───→ 시간(분)
  0   5   10  15  20  25  30  35

구간 설명:
[0-5분]   도입/수색 — 낮은 텐션, 세계관 흡수
[5-15분]  첫 심문 — 텐션 상승, NPC 성격 파악
[15-20분] 추가 수색/심문 — 밸리, 정보 정리 시간
[20-25분] 실언/핵심 단서 — 서프라이즈 피크
[25-30분] 증거 보드 — 사고 구간, 낮은 텐션이지만 높은 집중
[30-35분] 고발/해설 — 최종 카타르시스
```

### 2.2 턴 예산 (보통 난이도)

| 활동 | 예상 턴 수 | 비율 |
|------|:--------:|:----:|
| 장소 수색 | 12~18 | 30% |
| NPC 심문 | 20~30 | 45% |
| 증거 보드 조작 | 8~12 | 15% |
| 시스템 메시지/전환 | 5~8 | 10% |
| **총 턴** | **45~68** | 100% |

### 2.3 페이싱 제어 메카닉

```yaml
pacing_controls:
  # 너무 빨리 해결하는 경우 (쉬움 제외)
  anti_rush:
    condition: "수집 단서 < 60% 인데 가설 정합성 85%+"
    action: "미발견 중요 단서가 존재합니다" 힌트 표시
    reason: "최소한의 탐색 경험 보장"

  # 너무 오래 걸리는 경우
  anti_stall:
    condition: "30분 경과 + 가설 정합성 < 40%"
    action: |
      관찰 스킬 일시 강화 (+1턴)
      "조사관의 직감" 시스템 → 가장 유관 장소/NPC 하이라이트
    
  # 반복 심문 방지
  repetition_guard:
    condition: "같은 NPC에게 5턴 이상 연속"
    action: NPC → "이미 말한 내용인데요. 다른 분에게 물어보시는 건 어떨까요?"
```

## 3. 핵심 전환점 (Inflection Points)

게임 내에서 플레이 경험이 질적으로 변화하는 순간들:

| 전환점 | 트리거 | 체감 변화 | 설계 의도 |
|--------|--------|----------|----------|
| **첫 실언** | NPC anxiety ≥ 0.7 + 비밀 근접 질문 | "내 질문이 효과가 있다!" 에이전시 확인 | 심문 메카닉에 대한 보상감 |
| **가설 첫 60%** | 단서 3개+ 연결 | 증거 보드의 가치 인지, 추리 방향 확신 | 수색→추론 모드 전환 유도 |
| **레드 헤링 발견** | 가설에 모순 페널티 발생 | "이건 미끼였구나!" 긴장감 유지 | 맹신 방지, 비판적 사고 유도 |
| **교차 검증 성공** | 2명 NPC 진술 비교로 모순 발견 | 탐정으로서의 성취감 | 다중 심문의 전략적 가치 |
| **확신 고발** | 정합성 90%↑ | "드디어!" 카타르시스 빌드업 | 고발 순간의 극적 무게감 |
| **해설** | 고발 후 자동 | "아, 그런 거였구나" 논리적 쾌감 | 추리 과정의 정리 + 학습 |

## 4. 메타 진행 밸런싱

### 4.1 레벨업 곡선

```
필요 XP
  ↑
800│                                          ★ Lv.50
   │                                       ╱
600│                                    ╱
   │                               ╱
400│                          ╱
   │                    ╱
200│              ╱
   │        ╱
100│  ╱
   ├──┬──┬──┬──┬──┬──┬──┬──┬──┬──→ 레벨
   1  5  10 15 20 25 30 35 40 50

공식: XP_needed(level) = 100 × level^1.2

레벨 1→2:   100 XP (사건 1건)
레벨 5→6:   174 XP (사건 1~2건)
레벨 10→11: 263 XP (사건 2건)
레벨 20→21: 435 XP (사건 3건)
레벨 50:    누적 ~15,000 XP (약 120건)
```

### 4.2 XP 획득 구조

| 액션 | XP | 비고 |
|------|:--:|------|
| 사건 해결 (정답) | 100 | 기본 보상 |
| 동기 정답 | +30 | 보너스 |
| 수단 정답 | +30 | 보너스 |
| 빠른 해결 (<15턴) | +50% | 배율 |
| 빠른 해결 (<25턴) | +25% | 배율 |
| 모든 단서 발견 | +20 | 탐색왕 |
| 히든 이벤트 발동 | +15/건 | 각각 |
| 자백 유도 | +100 | 히든 엔딩 |
| 오판 | -20 | 패널티 |

### 4.3 스킬 트리 언락 순서 권장

```
                    [조사관 레벨]
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
    🔍 관찰 계열    🧠 심리 계열    🔬 법의학 계열
     Lv.2 기본       Lv.4 기본       Lv.6 기본
     Lv.8 중급       Lv.10 중급      Lv.14 중급
     Lv.16 고급      Lv.20 고급      Lv.25 고급
                         │
                         ▼
                   💡 추론 계열
                    Lv.12 기본
                    Lv.22 중급
                    Lv.30 고급

권장 빌드 경로:
- 초보: 관찰(기본) → 심리(기본) → 추론(기본) → 법의학(기본)
- 속공: 추론(기본) → 심리(고급) → 속도 빌드
- 탐색: 관찰(고급) → 법의학(고급) → 완전 수색 빌드
```

## 5. 감정 분석 파이프라인

NPC의 감정 상태와 플레이어 질문의 의도를 분석하는 시스템.

### 5.1 플레이어 질문 분류 체계

```yaml
question_classification:
  categories:
    - factual: "사실 확인 질문 (누가, 언제, 어디서)"
      example: "만찬은 몇 시에 시작했나요?"
      emotion_impact: +0.05 (미미)
    
    - relational: "관계/감정 관련 질문"
      example: "빅토리아와 사이가 좋았나요?"
      emotion_impact: +0.10~0.20 (대상에 따라)
    
    - accusatory: "추궁/비난 질문"
      example: "당신이 독을 넣은 거 아닌가요?"
      emotion_impact: +0.30 (급등)
    
    - empathetic: "공감/이해 질문"
      example: "힘드셨겠군요. 괜찮으세요?"
      emotion_impact: -0.15 (감소)
    
    - trap: "함정 질문 (이전 진술과 모순 유도)"
      example: "아까는 주방에 안 갔다고 하셨는데?"
      emotion_impact: +0.25 (만약 모순이면 +0.40)
```

### 5.2 LLM 감정 분석 프롬프트

```markdown
## Emotion Analysis Prompt

아래 NPC 대화 맥락을 분석하여 감정 상태를 JSON으로 출력하세요.

### 입력
- NPC 현재 감정: {current_emotion}
- NPC 비밀: {secret_summary}
- 플레이어 질문: {player_question}
- 질문 카테고리: {question_category}

### 출력 형식
```json
{
  "new_emotion": "anxious",
  "emotion_delta": +0.15,
  "slip_risk": 0.3,
  "response_tone": "defensive_but_polite",
  "nonverbal": "손가락으로 테이블을 두드린다"
}
```

### 규칙
- 비밀에 가까운 질문일수록 emotion_delta 증가
- empathetic 질문은 emotion_delta 음수 가능
- slip_risk는 0.0~1.0, 0.7 이상이면 실언 이벤트 트리거
- nonverbal은 NPC 페르소나의 감정 표현 목록에서 선택
```

## 8. UGC 도구 설계 (Phase 3+)

메타 진행 50건+ 해결 후 해금되는 **커스텀 시나리오 에디터**.

### 8.1 에디터 핵심 기능

```yaml
scenario_editor:
  mode: "guided"  # 완전 자유가 아닌 가이드 모드
  
  steps:
    1_setting: "세팅 선택 (기존 장소 풀에서)"
    2_crime: "범죄 유형 선택"
    3_characters: "NPC 수/프로필 기본 설정 (Big Five 슬라이더)"
    4_clues: "커스텀 단서 입력 (텍스트)"
    5_validate: "SMT 검증 — 풀이 가능한지 자동 체크"
    6_publish: "커뮤니티 공유 (선택)"
  
  constraints:
    - SMT 검증 통과 필수 (풀 수 없는 시나리오 방지)
    - 콘텐츠 필터 통과 필수
    - 최소 4명 용의자, 최소 5개 단서
  
  community:
    - 다른 유저의 시나리오 플레이 가능
    - 별점 + 리뷰 시스템
    - "이 달의 베스트 사건" 큐레이션
```
