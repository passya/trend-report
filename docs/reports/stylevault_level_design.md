## 3. 가격 밸런싱

### 3.1 작가 가격 설정 가이드

| 지표 | 참고 범위 | 근거 |
|------|----------|------|
| Indie 티어 | $19~$49/월 | 인디 개발자 월 에셋 예산의 5~10% |
| Studio 티어 | $99~$249/월 | 전통 일러스트 커미션 1건($200~$500) 비용과 동등 |
| Exclusive 티어 | $1,000~$5,000/월 | 풀타임 일러스트레이터 월급 대비 50~70% |
| 시장 정상가 | 상위 20% 작가 평균 × 0.8 | 초기 진입 장벽을 낮추기 위한 할인 |

### 3.2 플랫폼 수수료 밸런스

```
                    수수료 비교
    ┌──────────────┬──────────────────┐
    │ 플랫폼       │ 수수료 비율      │
    ├──────────────┼──────────────────┤
    │ App Store    │ 30%             │
    │ Steam        │ 30%             │
    │ Civitai      │ 0% (무료 공유)  │
    │ exactly.ai   │ ~40% (추정)     │
    │ StyleVault   │ 15% ✅          │
    │ 전통 에이전시  │ 20~50%         │
    └──────────────┴──────────────────┘
```

**15% 선정 근거:** 
- 10% 미만: 인프라 비용 회수 불가 (GPU 비용 구조)
- 20% 이상: 작가 이탈 리스크 (Civitai 0%와 비교됨)
- 15%는 "저작권 보호 + 마켓플레이스 노출"의 가치에 대한 적정 대가

## 4. 세션 페이싱

### 4.1 작가 감정 곡선 (첫 LoRA 생성 세션)

```
감정 (긍정↑ / 부정↓)

   ↑  │    ★스타일분석     ★QA통과        ★첫판매
      │    결과 확인       & 등록         알림!
   4  │      ╱╲             ╱╲             ╱╲
   3  │    ╱    ╲         ╱    ╲         ╱    ╲╲
   2  │  ╱      ╲       ╱      ╲       ╱      ╲╲───
   1  │╱         ╲    ╱         ╲    ╱
   0  │           ╲  ╱           ╲  ╱
  -1  │            ╲╱              ╲╱
      │          인증과정        가격설정
   ↓  │          번거로움        고민
      └──────────────────────────────────────────
        가입   인증   업로드  학습  QA  설정  등록  대기  판매
```

**설계 원칙:**
- 부정 구간(인증, 가격 고민) 직후 반드시 긍정 보상(분석 결과, QA 통과)
- 최대 부정 구간은 10분을 넘기지 않음
- "대기" 구간(학습 ~15분)에는 프로그레스바 + 학습 과정 시각화로 지루함 방지

### 4.2 턴 턴당 예산

| 단계 | 소요 시간 | 사용자 행동 | UI 가이드 |
|------|----------|-----------|----------|
| 가입 | 2분 | 소셜 로그인 | 원클릭 |
| 인증 | 5~10분 | 플랫폼 연동 + 확인 | 프로그레스바 3단계 |
| 업로드 | 5분 | 이미지 선택 + 드래그앤드롭 | 실시간 태그/점수 |
| 저작권 선언 | 2분 | 체크박스 + 확인 | 법률 용어 → 쉬운 설명 |
| LoRA 학습 | 15~20분 (비동기) | 대기 (알림) | 학습 진행률 + 예상 시간 |
| QA 확인 | 3분 | 리포트 + 데모 확인 | 점수 시각화 |
| 라이선스 설정 | 5분 | 가격/조건 입력 | 추천 가격 표시 |
| 스토어 등록 | 1분 | 최종 확인 클릭 | 미리보기 + 공유 링크 |
| **Total** | **~45분** (학습 대기 제외) | | |

## 5. QA 시스템 밸런싱

### 5.1 QA 점수별 분류

| QA 종합 점수 | 등급 | 마켓플레이스 노출 | 대응 |
|-------------|------|-----------------|------|
| 90~100 | ⭐ Premium | 상단 노출 + "고품질" 배지 | 자동 등록 |
| 80~89 | 🟢 Standard | 일반 노출 | 자동 등록 |
| 70~79 | 🟡 Basic | 노출 가능하나 하단 | 개선 가이드 제공 |
| 60~69 | 🟠 Needs Work | 등록 불가 | 구체적 개선 사항 + 무료 재학습 |
| 0~59 | 🔴 Rejected | 등록 불가 | 수동 리뷰 에스컬레이션 |

### 5.2 QA 세부 기준 & 가중치

| 항목 | 가중치 | 통과 기준 | 측정 방법 |
|------|:------:|----------|----------|
| 스타일 재현도 | 35% | ≥ 0.85 | CLIP cosine similarity (학습데이터 vs 생성물) |
| 프롬프트 반응성 | 25% | ≥ 0.80 | CLIP text-image alignment (다양한 프롬프트) |
| 다양성 범위 | 20% | ≥ 0.65 | 20개 프롬프트 출력물 간 벡터 분산 |
| 과적합 위험 | 15% | ≤ 0.25 | 학습데이터 vs 생성물 pixel similarity |
| 저작권 충돌 | 5% | 0건 | Pinecone ANN (기존 LoRA 유사도 < 0.92) |

  ─────────────────────────────
  순이익: $540.00 - $150.70 = $389.30
